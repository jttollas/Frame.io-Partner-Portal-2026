## Design Context

### Users
Software developers and technical partners building integrations on top of Frame.io's V4 API. Two primary audiences: (1) **Platform developers** — engineers building workflow, DAM, and creative tooling integrations who want fast access to API data and credentials; (2) **C2C partners** — hardware and firmware engineers at camera manufacturers who need device-level upload analytics and provisioning. Both audiences are technical and task-oriented. They come to the portal to build, debug, and monitor — not to browse.

### Brand Personality
Cinematic. Precise. Authoritative.

Modeled after **next.frame.io** — Frame.io's design language for the next-generation platform: deep-space dark backgrounds, confident serif/sans type pairing, a blue/teal accent palette that feels premium without being cold. The portal should feel like it belongs in the same product family — not a developer-afterthought, but a first-class surface.

### Aesthetic Direction
- **Theme**: Dark mode only. Deep navy-black backgrounds (`#060810`), layered surface cards (`#0c1019` → `#111623` → `#1a2033`), minimal but intentional use of glow effects.
- **Typography**: DM Serif Display for hero headings (editorial weight, cinematic), DM Sans for all UI text (clean, legible, developer-friendly). Monospace for code/credentials.
- **Color**: Primary accent `#4b8ef7` (blue), secondary `#0bc5ea` (teal for C2C), semantic green/amber/red for status. Gold/amber (`#f59e0b`) reserved for Premier tier — signals premium without being garish.
- **Motion**: Purposeful and restrained. Transitions should feel confident, not bouncy. Honor `prefers-reduced-motion`.
- **References**: next.frame.io + next.developer.frame.io — cinematic, dark, editorial, developer-native. Stripe — typographic precision and data clarity. Vercel — dark-mode developer platform done right.
- **Anti-references**: Generic SaaS dashboards (Bootstrap/Material overuse). Startup playfulness (too many gradients and illustrations). Legacy enterprise density (Jira-like tables and chrome).

### Design Principles

1. **Cinematic confidence** — Every screen should feel like it belongs in the Frame.io product family. Dark, immersive, and precise. No generic UI chrome.

2. **Data earns its place** — Dashboards are dense by necessity. Every number, chart, and table should serve a clear developer job-to-be-done. Remove anything decorative that doesn't inform.

3. **Tier clarity without friction** — The three-tier system (Developer / Partner / Premier) should communicate access levels in a way that feels aspirational, not punishing. Locked sections should make users want to upgrade, not feel excluded.

4. **Typography does the heavy lifting** — The DM Serif Display / DM Sans pairing creates hierarchy without needing decorative elements. Let type scale, weight, and contrast do the work.

5. **Developer trust first** — Credentials, API keys, and usage data are high-stakes. The UI must feel secure, organized, and reliable. Nothing should feel accidental or cluttered in sections handling sensitive data.
